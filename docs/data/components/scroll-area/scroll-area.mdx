## Anatomy

```jsx
<ScrollArea.Root>
  <ScrollArea.Viewport />
  <ScrollArea.Scrollbar orientation="vertical">
    <ScrollArea.Thumb />
  </ScrollArea.Scrollbar>
  <ScrollArea.Scrollbar orientation="horizontal">
    <ScrollArea.Thumb />
  </ScrollArea.Scrollbar>
  <ScrollArea.Corner />
</ScrollArea.Root>
```

## Type

The scrollbars can be either `overlay` (default) or `inset`. The `overlay` type will overlay the scrollbar on top of the content, while the `inset` type will push the content to make space for the scrollbar.

```jsx
<ScrollArea.Root type="inset">
```

### Gutter stability

When using `inset` scrollbars, the `gutter` prop can be used to keep a gutter present when the scrollbar is hidden. This is useful to prevent layout shift when the scrollbar is shown or hidden based on the content.

Possible values match the CSS `scrollbar-gutter` property:

- `stable` (default; padding added to account for vertical scrollbar)
- `both-edges` (symmetrical padding added for vertical scrollbar)
- `none` (layout shift allowed)

```jsx
<ScrollArea.Root gutter="both-edges">
```

## Styling

The scrollbar elements can be shown conditionally based on the user's interaction with the Scroll Area. The `[data-scrolling]` and `[data-hovering]` attributes are added to the scrollbar element while the user is scrolling or hovering over the Scroll Area.

```css
.ScrollAreaScrollbar[data-scrolling],
.ScrollAreaScrollbar[data-hovering] {
  opacity: 1;
}
```

### Corner

When using `inset` scrollbars, the scrollbar elements can prevent overlapping by rendering a `Corner`:

```jsx
<ScrollArea.Scrollbar orientation="vertical" />
<ScrollArea.Scrollbar orientation="horizontal" />
<ScrollArea.Corner />
```
